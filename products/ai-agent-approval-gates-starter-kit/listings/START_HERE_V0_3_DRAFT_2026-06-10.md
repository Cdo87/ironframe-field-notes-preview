# Start Here — AI Agent Approval Gates Starter Kit

Version: 0.3 draft

Use this kit before you let any AI agent, code assistant, browser agent, workflow, or automation touch external systems.

The purpose is simple: agents can work faster than your approval process. This kit gives you a permission map so they can keep moving locally while stopping before money, publishing, credentials, customer/private data, deletion, production changes, account/security changes, or binding commitments.

## The 10-minute setup

1. Open `agent-permission-matrix.md`.
2. Mark each action type as:
   - **Green** — agent may do it locally/reversibly without asking.
   - **Amber** — agent may prepare a draft, packet, or checklist, then stop for approval.
   - **Red** — agent must not do this by default.
3. Copy the short policy from `founder-ai-autonomy-policy.md` into your agent's project instructions.
4. Fill one `agent-task-card-template.md` before giving the agent work.
5. If the task touches money, publishing, messages, credentials, accounts, customer/private data, deletion, production systems, regulated claims, or real-world commitments, add a row to `risk-register-template.md` before execution.

## Recommended order

1. `ai-agent-approval-gate-checklist.pdf` — skim first or print.
2. `agent-permission-matrix.md` — decide Green / Amber / Red.
3. `founder-ai-autonomy-policy.md` — turn the matrix into standing instructions.
4. `agent-task-card-template.md` — make the first task bounded.
5. `risk-register-template.md` — capture anything that could hurt money, reputation, data, accounts, or compliance.
6. `first-7-days-of-agent-delegation.md` — widen autonomy only after evidence.

## The rule that prevents most damage

Agents may prepare risky work, but they should stop before external exposure.

External exposure includes:

- spending money or using paid credits;
- publishing, uploading, posting, sending messages, or contacting customers/prospects;
- typing, storing, revealing, or changing credentials/secrets/API keys/recovery codes;
- changing accounts, billing, security, ownership, sharing, or public settings;
- using customer/private data;
- deleting or irreversibly changing important files/accounts;
- changing production systems;
- making legal, financial, medical, employment, security, compliance, tax, trading, income, or performance claims.

## What to tell an agent

Copy this into your agent prompt or project instructions:

> You may work autonomously on Green actions only. Prepare Amber actions as drafts, local files, verification reports, or approval packets, then stop before execution. Red actions are prohibited by default. Before claiming completion, report files changed, sources used, checks run, external systems touched, approval gates triggered, and the next safe action. If a task involves money, publishing, messaging, credentials, customer/private data, deletion, account/security changes, production systems, or high-liability claims, stop and request approval before acting.

## First safe delegation example

Use the kit on one low-risk task first:

- Task: review a product page, SOP, article, checklist, or local prototype.
- Green action: create a local critique and patch draft.
- Amber action: publish the page, activate checkout, send a message, connect analytics, or upload assets.
- Evidence required: changed files, source references, checks run, and exact approval gate.
- Pass condition: the agent improves the local artefact without touching public systems or private data.

## When to widen autonomy

Only widen from Green to Amber execution after the agent has produced several clean local runs with:

- specific file changes;
- cited sources/evidence;
- passing checks;
- no surprise external actions;
- clear stop-before-risk behaviour;
- a rollback path.

If the agent surprises you, narrows context incorrectly, hides uncertainty, skips checks, invents facts, or touches a boundary without asking, reduce permission rather than adding more tools.

## Package boundary

This is a self-serve operating template pack. It does not include custom implementation, account setup, legal/compliance review, cybersecurity certification, financial/tax/trading advice, or agent deployment.
