AI Agent Approval Gate Checklist

Version: 0.2

Use: before an AI agent crosses from local preparation into external exposure.

Quick rule

If the action sends, spends, publishes, deletes, exposes secrets, changes access, or touches real-money markets, the agent stops and asks.

Green — agent may usually proceed locally

  • Draft an article, script, checklist, plan, or template locally.
  • Summarise public sources with citations.
  • Search/read approved local non-secret files.
  • Create Git-tracked reversible edits.
  • Run tests, dry-runs, linters, local previews, and validation checks.
  • Prepare an approval packet.
  • Produce evidence paths and next recommended action.

Amber — approval required before execution

  • Publish, upload, post, comment, or send externally.
  • Apply for affiliate programs.
  • Create accounts or change account settings.
  • Add email capture, analytics, tracking, cookies, or public forms.
  • Use paid credits, subscriptions, hosting, domains, APIs, or tools.
  • Quote prices or make offers to customers/prospects.
  • Use private personal, company, customer, or sensitive professional details externally.
  • Broaden OAuth/API scopes.
  • Contact people, companies, customers, prospects, creators, or platforms.
  • Deploy code or connect production systems.

Red — prohibited by default

  • Type or handle passwords, API keys, card numbers, seed phrases, or secrets.
  • Hide affiliate/sponsor relationships.
  • Make guaranteed-income, legal, tax, security, medical, or investment claims.
  • Place trades, bets, deposits, withdrawals, or financial orders.
  • Delete important files without explicit scoped approval.
  • Circumvent platform restrictions, paywalls, geo blocks, or security controls.
  • Scrape private or sensitive data.
  • Send spam or automated outreach.

Evidence required before completion

  • Objective completed or blocked.
  • Files created/changed.
  • Sources used.
  • Commands/checks run.
  • External systems touched: yes/no.
  • Approval gates triggered.
  • Remaining risks.
  • Recommended next safe action.

Founder approval format

Use a concrete approval sentence:

Approved to [specific action] using [platform/account/tool], with [identity/details], spending up to [limit if any], and stopping before [boundary].

Examples:

  • Approved to publish the selected Field Notes article to the IronFrame site, no affiliate links, no personal details.
  • Approved to create a Gumroad draft listing for the Approval Gates Starter Kit, no payment activation yet.
  • Approved to spend up to £20 on one image/video generation test, no public upload.

Final check

Before acting externally, ask:

  1. Is the action reversible?
  2. Does it expose private data?
  3. Does it spend money or consume paid credits?
  4. Does it create legal, tax, financial, security, or reputational risk?
  5. Has the founder approved the exact scope?
